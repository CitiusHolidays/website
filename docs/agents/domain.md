# Domain Docs

This is a single-context repository.

## Before exploring

- Read the root `CONTEXT.md`.
- Read ADRs under `docs/adr/` that affect the area being changed.
- Proceed silently if a referenced domain document does not exist.

## Vocabulary

Use the canonical terms defined in `CONTEXT.md` in specifications, ticket titles, tests, and implementation discussions. Avoid synonyms that the glossary explicitly rejects.

If a needed concept is absent, reconsider whether it belongs to the domain model or capture the gap through the domain-modeling workflow.

## ADR conflicts

Surface any proposed behavior that contradicts an existing ADR rather than silently overriding it.
